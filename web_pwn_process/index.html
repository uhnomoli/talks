The Process

  1. Identify goal
    • The what
       ⤿ (flag or access)
  2. Identify attack vector
    • The who
       ⤿ (server or person)
  3. Enumerate
    • The how
       ⤿ (find the bug)
  4. Exploit
    • The why
       ⤿ (be 1337 hacker man)

The goal

  • CTF
    • SCENARIO: A flag
       ⤿ Where is it
       ⤿ (cookie, env, file)
  • HTB / OffSec
    • SCENARIO: Access
       ⤿ To what
       ⤿ (creds, rce)
  • What we're trying to accomplish often dictates vulnerability and can drastically narrow our search
    • Steal a cookie => xss
    • Read a file => lfi, rce
    • Steal creds => lfi, sqli
  • Flag: do we know it's location (e.g. Dockerfile, source, description, etc.)

The attack vector

  • Person (interactive)
    • Pretty much always xss
    • Maybe cron job
    • Maybe phishing
  • Server (non-interactive)
    • Everything else

Enumerate

  • The hard part
     ⤿ (easier after previous steps)
     ⤿ (and experience)
  • Remember the goal
  • Identify technology stack
     ⤿ (static or dynamic)

Exploit

  • Don't forget the dumb things
